# 多源勘察数据联动展示与证据链追溯系统

> **空天地多源勘察数据融合展示与风险证据链决策平台**
>
> 把 **正射影像、三维点云、物探剖面、钻孔柱状图、勘察报告** 统一挂到同一个工程位置上：
> 看得见数据 → 找得到位置 → 连得上证据 → 解释得清风险 → 给得出建议。

---

## ✨ 核心能力

| 能力 | 说明 |
|---|---|
| 🗺️ **多源空间联动** | 正射影像底图 + 线路轴线 + 钻孔点 + 物探测线 + 风险区，统一在工程坐标系下 |
| 🏔️ **三维点云/地形** | 基于 DEM 生成的坡度着色点云，选中风险区自动飞行定位并高亮边界 |
| 🔗 **证据链追溯** | 每个风险对象挂载 5 类证据（影像/点云/物探/钻孔/报告），可逐条追溯 |
| 💬 **智能对话引擎** | 自然语言交互：定位/查询/对比/解释/报告，多轮上下文记忆，对话直接驱动地图联动（**完全离线，无需 API**） |
| 🔍 **条件查询** | "坡度大于 30 度的风险""水位浅于 3 米的钻孔""K12+300 到 K12+800 之间的风险" |
| 📑 **多格式报告生成** | **Word(.docx)/Markdown/HTML** 三格式 × 单风险/全线综合两范围，Word 内嵌物探剖面与钻孔柱状图 |
| 📊 **多维证据可视化** | ECharts 风险雷达图（6 维度评分）+ 多风险对比叠加 + 钻孔地层条形图 |
| 📏 **里程轴联动** | K12+000~K13+000 里程轴，风险按位置落点，点击即定位 |

## 🎯 系统定位（不是普通可视化大屏）

普通展示只回答"**有什么数据**"。本系统回答"**这个风险为什么成立**"：

- **坐标**解决"在哪里"（影像/点云/钻孔落在同一空间）
- **里程**解决"工程上怎么描述"（K12+380 左侧边坡上方）
- **对象**解决"它是什么问题"（洞口边坡失稳 / 富水破碎带）
- **证据表**把五类数据串成一条可追溯的链

## 🚀 快速开始

### 一键运行（Windows）

双击 **`run.bat`** 即可。脚本会自动：
1. 检查并安装 Python 依赖（首次约 30 秒）
2. 生成多源勘察样例数据（首次约 10 秒）
3. 启动 Web 服务并自动打开浏览器

> 浏览器访问：**http://localhost:8000**

### 手动运行

```bash
# 1. 安装依赖
pip install fastapi uvicorn numpy pandas matplotlib pillow

# 2. 生成数据（仅首次）
python backend/generate_data.py

# 3. 启动服务
python -m uvicorn backend.app:app --host 0.0.0.0 --port 8000

# 4. 浏览器打开 http://localhost:8000
```

### 环境要求

- **Python 3.8+**（推荐 3.9 / 3.10）
- 浏览器：Chrome / Edge（需支持 WebGL）
- 无需联网 —— 前端三方库（Leaflet / Three.js / ECharts）已本地打包

## 💬 智能对话引擎（核心交互能力）

底部「智能对话」标签是一个**完全离线的自然语言交互引擎**（`backend/nlu.py`），无需任何外部大模型 API，比赛现场断网可用。

### 五类意图 + 自然语言操作

| 意图 | 示例指令 | 系统动作 |
|---|---|---|
| 🗺️ **定位** | "带我去看看 K12+380 的边坡" | 自动飞行定位 + 证据链联动 + 3D 聚焦 |
| 🔍 **查询** | "坡度大于 30 度的风险有哪些" / "水位浅于 3 米的钻孔" | 结构化条件过滤，返回符合条件的风险/钻孔 |
| 📊 **对比** | "R001 和 R002 哪个风险更高" | 生成对比表 + 结论 |
| 💡 **解释** | "K12+720 富水破碎带如何处理" | RAG 检索 + 综合解释 + 设计建议 |
| 📑 **报告** | "给 R001 生成一份报告" | 自动切到报告页并生成 |

### 关键特性

- **多轮上下文记忆**：连续对话保留状态。"带我去 K12+380" → "那它的截排水怎么设计"，"它"自动指代 R001
- **动作联动地图（Function Calling 本地实现）**：对话中识别出定位/报告意图时，自动触发地图飞行、证据链填充、报告生成
- **证据可追溯**：每个回答附带可点击的「证据来源」卡片，点击即跳转对应风险区
- **RAG 检索**：报告/风险/钻孔文本切成语料，TF-IDF 向量化（字符级 n-gram，对中文友好），召回相关片段

### 支持的查询语法

```
坡度大于 30 度的风险          → slope_gt=30
坡度小于 15 度的区域          → slope_lt=15
地下水位浅于 3 米的钻孔       → water_lt=3
电阻率小于 200 的风险         → rho_lt=200
K12+300 到 K12+800 之间的风险  → 里程范围查询
R001 和 R002 哪个风险更高      → 对比分析
```

## 📑 多格式报告生成引擎

底部「报告生成」标签可产出**正式工程交付文档**（`backend/report_gen.py`），完全本地生成，无需 Office/WPS。

### 三种格式 × 两种范围

| | 单风险报告 | 全线综合报告 |
|---|---|---|
| **Word (.docx)** | 封面 + 多源证据表 + 参数 + 解释 + 建议 + 钻孔地层表 + **嵌入物探/柱状图** | 工程概述 + 风险统计 + 逐风险分析 + 对比 + 总体结论 + **嵌入全部图件** |
| **Markdown (.md)** | 同上（纯文本） | 同上（纯文本） |
| **HTML (.html)** | 同上 + base64 内嵌图（浏览器可直接打印 PDF） | 同上 + 全部图件 |

### 使用方式

**界面操作**：底部「报告生成」标签 → 选范围（全线/单风险）→ 选格式（Word/MD/HTML）→ 点「生成预览」在线查看 → 点「下载文件」获取正式文档。

**对话触发**：在智能对话中输入「给 R001 生成一份报告」→ 自动切换到报告页并生成。

**API 调用**：
```
GET /api/report/preview?scope=full              # 全线报告预览
GET /api/report/download?scope=risk&rid=R001&fmt=docx   # 下载单风险 Word
```

### Word 报告结构（答辩可演示）

一份完整的全线综合 Word 报告包含：
- **封面**：项目名、报告类型、生成时间、生成方式
- **一、工程概述**：线路参数、钻孔/物探/风险统计表
- **二、风险统计**：全部风险一览表
- **三、逐风险多源证据分析**：每个风险含基本信息表、多源证据表、关键参数表、**嵌入物探剖面图**、**嵌入钻孔柱状图**、综合解释、设计建议、地层表
- **四、风险对比分析**：跨风险参数对比表 + 首要风险结论
- **五、总体结论与建议**：分优先级处置建议

> 实测：全线 Word 报告 243KB，含 15 个表格、7 张嵌入图片、67 个段落。

## 📊 多维证据可视化 (ECharts)

系统在证据链面板集成了三种 ECharts 可视化，把"数字"变成"图形证据"：

### 1. 风险雷达图（单风险）

选中任一风险区，证据面板顶部自动渲染 **6 维度雷达图**，把多源证据量化为 0-100 分：

| 维度 | 评分依据 |
|---|---|
| 地形坡度 | 最大坡度 / 45° × 100 |
| 高差起伏 | 相对高差 / 60m × 100 |
| 物探异常 | (1000 - 最低电阻率) / 900 × 100 |
| 钻孔揭露 | 风化层厚度 + (100-RQD) 加权 |
| 地下水 | (10 - 水位埋深) / 10 × 100 |
| 综合等级 | 高=90, 中高=70, 中=50 |

不同风险类型的雷达形状特征鲜明：**边坡风险**（R001）在坡度/高差维度突出，**富水风险**（R002）在物探/地下水维度突出 —— 一眼就能看出致险机理差异。

### 2. 多风险对比雷达（对话触发）

在智能对话中问「R001 和 R002 哪个风险更高」，系统自动在证据面板渲染**多风险叠加雷达图**，直观对比不同风险的多维特征。

### 3. 钻孔地层条形图

证据面板的钻孔卡片中，每个钻孔自动渲染 **ECharts 横向条形图**，按岩性着色展示地层分布，鼠标悬停显示层厚与描述 —— 比单纯看柱状图图片更易扫读。

## 🛠 技术栈

- **前端**：原生 HTML/CSS/JS + Leaflet（2D 地图）+ Three.js（3D 点云）+ ECharts + marked.js
- **后端**：Python FastAPI + uvicorn
- **数据生成**：NumPy + matplotlib + Pillow（DEM/影像/点云/剖面/柱状图）
- **存储**：JSON / PNG / PLY / CSV 文件（轻量，无需数据库）

## 🎨 设计原则

1. **离线可用** —— 所有前端库本地打包，断网也能演示
2. **证据可追溯** —— 任何 AI 回答都附带可点击的证据来源
3. **轻量自洽** —— 数据由生成器一次性产出，逻辑闭环，不依赖外部服务
4. **演示友好** —— 一键启动，3 个典型案例覆盖核心风险类型

## ❓ 常见问题

**Q: 想重新生成数据？**
A: 删除 `backend/data` 文件夹，重新运行 `python backend/generate_data.py` 或 `run.bat`。

**Q: 想换一个工程案例？**
A: 修改 `backend/generate_data.py` 中的 `RISK_OBJECTS`、`BOREHOLES`、`GEOPHYSICS_LINES`、`SURVEY_REPORT` 后重新生成。

**Q: 端口被占用？**
A: 修改 `run.bat` 或启动命令中的 `--port 8000` 为其他端口。

**Q: 3D 点云看不到？**
A: 确认浏览器支持 WebGL（Chrome/Edge 均可）；点云首次加载约 2-3 秒。
